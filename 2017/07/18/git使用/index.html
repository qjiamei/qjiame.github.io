基本使用

  1. git config –global user.name “que”

  2. git config –global user.name.email “que@163.com“

  3. git config –global user.password “123”

  4. git config –list

  5. git init

  6. git clone http://XXX

  7. git add aa.md

  8. git commit -m ‘描述’

  9. git push origin master (推上去)

sea维护资料使用命令

  1. git init

  2. git clone http:XXX

  3. git checkout -b 0715_master_que (建议大家都使用自己的分支上管理，日期+master+名字)

  4. git add -A love (可以一次性管理全部文件夹)

  5. git commit -m ‘改动的描述’

  6. git checkout master (切换到主分支)

  7. git pull origin master (使用之前先拉一下)

  8. git merge 0715_master_que(跟主分支融合)

  9. git push origin master(推到主分支上)

在分支上使用一

  1. 使用 git checkout -b 0713bug_master

    • 这个命令相当于两个命令

    • git branch 0731bug_master (创建分支)

    • git checkout 0730bug_master (新建分支)

  2. 在新切分支上修改

    • git status (可查看哪些文件被修改)

    • git add bb.md (选择本次提交文件,多个路径用空格隔开)

    • git commit -m ‘加描述’

  3. 假设多次提交后,要merge到主分支上去

    • git status (先查看一下)

    • git stash (将这些改动的文件暂存起来)(git stash apply 切换回分支后使用此命令可以来到工作的点,改动过的都在)

    • git checkout master (切换到主分支上)

    • git pull origin master (先拉)

    • git merge 0731bug_master (融合)

    • git push origin master (推上去吧)

在分支上使用二

如果主分支有两个,比如一个稳定版,一个开发版,要merge到开发版应该如何操作呢?

  1. git log (在当前分支0713bug_master找一个commitId,这个commitId是当前所要merge的第一次提交的前一个commitId)

  2. git checkout develop (切换到开发版分支)

  3. git pull origin develop

  4. git checkout 0713bug_master (切换到自己的bug分支)

  5. git rebase –onto=develop commitId

  6. git checkout develop (查看是否有冲突,有则解决)

  7. git push origin develop

在分支上使用二(通用)

  1. git log(在当前分支0713bug_master上，找到要merge的commitId)

  2. git checkout develop

  3. git pull origin develop

  4. git cherry-pick commitId

查看是否有冲突,有则解决冲突

  1. git push origin develop

关于github上克隆项目的问题

  1. 新电脑需要先把公钥复制到 New SSH key里.

    • ssh key：ssh-keygen -t rsa -C “youremail@example.com“ (这里要一路回车)

    • 在用户目录下的.ssh文件夹中有id_rsa(私钥)和id_rsa.pub(公钥)

    • git clone git@XXx

    • 如果报错的话,有可能是网络环境问题,请在clone后+http协议地址.

    • git remote show origin (查看某个远程仓库的详细信息)

删除命令

  1. rm aa.md

  2. git rm aa.md

  3. git commit -m ‘描述’

  4. git push origin master

其它常用命令

  1. git branch (查看本地所有的分支)

  2. git add -A abc (abc为文件夹，文件夹下所有文档都将add)

  3. git add . 也是添加代码进去

  4. git branch -D 0713bug_master (强制删除0713bug_master分支)

  5. git reset –hard commitId(回滚到某次commitId状态，git reset –soft HEAD~ )

  6. git reset [–hard|soft|mixed|merge|keep] [commit|HEAD]

  1. git remote add kh https://git.oschina.Net/zgj2016/KHPro.git （这行命令执行一次就行，是为了给自己仓库的地址起个别名，下次提交不需要执行此行代码）

  2. git mv game.py wordgame.py(在git里重命名需要用到git mv 命令)

  3. echo *.temp > .gitignore （创建带点的忽略文件夹，并忽略.temp文件）

  4. 成功合并后要查看版本状态,这三命令

    • git status

    • git log --graph -all

    • git log --graph --pretty=oneline --abbrev-commit这个是只关心分支线的情况

  5. 恢复误删分支

    • git log –branches=”被删的分支名”

    • git branch develop 版本id

5.其它操作

  • git clone -b [分支名称] git@….git 克隆某一个分支

  • git pull <远程主机名> <远程分支名>:<本地分支名> 取回远程分支并合并到本地分支

  • git fetch origin <远程主机名> <远程分支名>:<新分支名> 取回远程分支并重命名未新分支

  1. LF警告

    • rm -rf .git // 删除.git

    • git config –global core.autocrlf false //禁用自动转换

    • git init //初始化git库

    • git add –all //提交所有修改到暂存区

  1. cloud not found(克隆的时候找不到)

    • 找不到存储库，最核心的问题是证书管理器保存的信息于当前拉取的信息有问题，所以执行一下操作

    • git credential-manager uninstall

    • git credential-manager install

六.版本回退

  1. git log命令显示从最近到最远的显示日志
  2. 如果嫌上面显示的信息太多的话，我们可以使用命令 git log –pretty=oneline 演示如下
  3. 我想把当前的版本回退到上一个版本
    git reset --hard HEAD^
  4. 如果要回退到上上个版本
    git reset --hard HEAD^^
  5. 如果要回退到前100个版本的话
    git reset --hard HEAD~100
  6. 通过版本号回退
    • 先查看版本号
    • 使用git reset --hard 版本号回退

七.Git撤销修改和删除文件操作

  • git checkout --readme.txt 意思就是，把readme.txt文件在工作区做的修改全部撤销，这里有2种情况，如下：
  1. readme.txt自动修改后，还没有放到暂存区，使用 撤销修改就回到和版本库一模一样的状态。
  2. 另外一种是readme.txt已经放入暂存区了，接着又作了修改，撤销修改就回到添加暂存区后的状态。
